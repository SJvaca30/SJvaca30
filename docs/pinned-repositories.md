# Pinned Repositories Strategy

GitHub profile pins are managed from the profile page through **Customize your pins**. GitHub supports pinning public repositories and gists that represent your work or contributions.

Recommended order for this profile:

| Priority | Repository | Why pin it |
| ---: | --- | --- |
| 1 | `expo/expo` | Recognizable mobile framework; two accepted documentation PRs. |
| 2 | `openrewrite/rewrite` | Strongest technical accepted contribution so far: formatter behavior fix. |
| 3 | `spring-projects/spring-boot` | Recognizable Java ecosystem project; accepted documentation PR plus one test PR in review. |
| 4 | `facebook/react-native` | Recognizable mobile ecosystem project; approved and closed with the project's `Merged` label. |
| 5 | `vercel/ai` | AI SDK provider behavior work currently in review. |
| 6 | `OpenAPITools/openapi-generator` | Generator/tooling behavior work currently in review. |

Avoid pinning forked personal copies of upstream repositories unless the fork itself contains a clear original project or showcase. The profile should emphasize accepted upstream work, not just the existence of local forks.

Official GitHub guide: [Pinning items to your profile](https://docs.github.com/en/account-and-profile/setting-up-and-managing-your-github-profile/customizing-your-profile/pinning-items-to-your-profile)
